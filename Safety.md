There are different types of safety [(1)](https://www.youtube.com/watch?v=I8UvQKvOSSw)
- [[Type Safety]]
- [[Resource Safety]]
- [[Memory Safety]]

Safety also means the absence of errors, such as [(1)](https://youtu.be/I8UvQKvOSSw?t=635):
- Logic errors.
- Resource leaks.
- Concurrency errors.
- Memory corruption.
- Type errors.
- Overflows and unanticipated conversions.
- Timing errors.
- Allocation unpredictability.
- Termination errors.

Static analysis and rules are required to eliminate errors [(1)](https://youtu.be/I8UvQKvOSSw?t=2060).
Rules are difficult to follow, so we need libraries that guide us.
